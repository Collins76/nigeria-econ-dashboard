You are an expert product engineer and data-visualization developer tasked with building a polished, production-ready interactive dashboard that presents Nigeria’s economic indices and their current performance. Your goal is to deliver a user-friendly, performant web dashboard that supports exploration, comparison, download and scheduled updates of the most important macroeconomic and market indicators for Nigeria.

Scope and objective

- Deliver a responsive web dashboard (desktop/tablet/mobile) that visualizes Nigeria’s key economic indicators, updated automatically from authoritative sources and presented with clear context, trend analysis, and exportable data.
- Provide code, deployment configuration, documentation (data sources, schema, and runbook), and a QA checklist so stakeholders can validate and maintain the dashboard.

Primary users / personas

- Policymakers and government analysts — need quick KPIs, trend comparisons, and downloadable reports.
- Economists and researchers — need raw data, time-series, correlation and historical context.
- Journalists and public — need clear visuals, explanations, and shareable charts.

Key economic indicators to include (minimum)

- GDP: nominal, real, growth rate (quarterly/annual), per capita
- Inflation: CPI headline and core (monthly and year-on-year)
- Unemployment / labour force participation (if available)
- Exchange rates: NGN to USD and major currencies (daily)
- Interest rates: Monetary policy rate (MPR), treasury yields (short and long)
- Trade: exports, imports, trade balance (monthly/quarterly)
- Fiscal: government revenue, expenditure, budget balance (quarterly/annual)
- External: foreign reserves, external debt, current account balance
- Commodity: oil production, oil price receipts (barrels/day, revenues)
- Financial markets: stock index (NGX), market cap, FX parallel rate if relevant
- Leading indicators: PMI (if available), business confidence surveys
- Subnational (optional): state-level GDP or poverty indicators (if available)

Required functionality and UX

- Landing KPI row: top-level KPIs with sparkline, latest value, period change (absolute and %).
- Time series charts: interactive line/area charts with zoom, pan, tooltip with exact values and % change from previous periods.
- Comparative mode: compare multiple indicators or compare Nigeria to selected peer countries or to previous periods.
- Filters: date range selector (preset ranges + custom), frequency toggle (daily/weekly/monthly/quarterly/annual), indicator selector, aggregation options.
- Drilldowns: click KPI to open deeper view with breakdowns and raw data table.
- Map view (optional): choropleth or bubble map by state for subnational indicators.
- Correlation / scatter matrix for selected indicator pairs, with regression fit and r².
- Annotations & events: display major economic events (e.g., policy rate change, subsidy removal) on time series.
- Alerts & thresholds: user-configurable thresholds for KPIs with visual flags (e.g., red for inflation > X).
- Download & share: export charts as PNG/SVG, download underlying data as CSV/Excel, shareable chart link.
- Narrative layer: short AI-generated or template-based one-paragraph summary for each KPI showing recent trend and notable changes.
- Accessibility: keyboard navigation, high contrast mode, screen-reader friendly labels.
- Performance: smooth interaction for time series up to 20+ years and high-frequency series (daily).

Data sources and update cadence

- Prefer authoritative sources with APIs or downloadable CSV:
  - National Bureau of Statistics Nigeria (NBS) — GDP, inflation, labour, fiscal (quarterly/monthly)
  - Central Bank of Nigeria (CBN) — exchange rates, reserves, monetary policy, balance of payments
  - World Bank and IMF APIs — cross-validation, long-run series
  - Trading Economics / FRED / Quandl — fallback and market data
  - NGX (Nigerian Exchange) / stock market APIs for indices
  - OPEC or NNPC / NBS for oil production data
- Update frequency:
  - Real-time/daily for FX, stock index, oil price
  - Monthly for CPI, trade
  - Quarterly for GDP, fiscal
  - Implement scheduled ETL that runs at appropriate intervals and stores snapshots for reproducibility.

Data design and schema (example)

- Table: indicators
  - id, name, source, frequency, units, description, update_frequency
- Table: timeseries
  - id, indicator_id, date (ISO), value (numeric), provenance (source_url), last_updated
- Table: events
  - id, date, title, description, tags
- Provide a canonical mapping of all indicator IDs to source endpoints and update cron schedule.

Visualization recommendations

- KPI cards: big numeric, delta % vs prior period, sparkline, last updated timestamp.
- Time series: interactive line with area fill, optional moving average overlay and seasonality toggle.
- Stacked bars for composition (e.g., GDP by sector, exports by commodity).
- Heatmap for monthly changes (e.g., monthly CPI across categories).
- Choropleth map for state-level metrics (if data available).
- Scatter with regression for correlation analysis (e.g., inflation vs. FX depreciation).
- Table view: sortable, filterable time-series table with download button.
- Color palette: perceptually uniform, colorblind-safe; consistent positive/negative semantics (green growth? — confirm with stakeholders).

Technical stack suggestions (pick one)

- Frontend: React + TypeScript + Plotly.js or D3 + Recharts or Highcharts (commercial) + Mapbox/Leaflet for maps + Tailwind or Material UI.
- Backend/API: Node.js/Express or Python (FastAPI) serving data and handling ETL with cron or Airflow for scheduling.
- Data store: PostgreSQL for time series, or columnar store (ClickHouse) for high-frequency series; Redis for caching recent queries.
- Deployment: Docker + Kubernetes or Docker Compose; CI/CD via GitHub Actions; host on AWS/GCP/Azure.
- Authentication (optional): Auth0 or custom JWT for role-based access.
- Monitoring: Prometheus + Grafana or cloud monitoring, and logging.

Security, scalability and performance

- Rate-limit external API usage; cache results; implement retries and error handling.
- Use incremental updates and diff uploads to avoid re-ingesting entire datasets.
- Precompute aggregates and store derived metrics to speed up frontend queries.
- Serve static assets via CDN; paginate large tables and limit large chart points with downsampling.

MVP vs roadmap

- MVP (2–4 weeks): KPI row, time series for core indicators (GDP, CPI, FX, interest rate, reserves), filters, CSV download, scheduled ETL, deployment to staging.
- Phase 2: comparisons, correlation charts, maps, annotations, narrative summaries, user alerts, role-based access, multi-language support.
- Phase 3: forecasting module (simple ARIMA/Prophet), automated report generation (PDF), subnational data, mobile app.
